Azure Windows VM 네트워크 연결 확인

2020-12-11
2020-12-11
Azure, Cloud, Windows

이번에는 Azure VM안에서의 네트워크 통신을 확인하는 방법을 포스트 하겠습니다.

먼저 Azure 환경에 아래와 같이 동일한 Vnet에 2대의 vm을 만들었습니다.

10.0.0.4 에서 10.0.0.5로 8080 포트로 연결이 정상적으로 되는지 확인하려고 합니다.
사전에 10.0.0.5 서버에서 IIS를 설치한 이후에 listen port를 8080으로 변경하였습니다.
IIS 설치 및 셋팅 관련 포스트는 추후 다시 올리도록 하겠습니다.

포트 체크를 telnet이나 ping으로 하시는 분들이 많은데 저는 아래의 powershell 명령어로 체크하겠습니다.

음.. 아래와 같이 TcpTestSucceded:False가 나옵니다.

  1. 윈도우 방화벽 확인하기
    window 접속하신후에 ‘Window firewall with Advanced Security’ 창을 엽니다.

왼쪽 사이드바에서 ‘Inbound Rule’을 오른쪽 마우스로 클릭하여 룰 추가를 클릭합니다.
아래의 화면이 나타나면 ‘Port’를 클릭하고 next를 클릭합니다.

Protocol(TCP)선택 후에 open할 포트를 입력합니다. 여기서는 8080을 입력합니다.

Allow, Block을 선택하는 창입니다. 저희는 허용정책을 만들어야 하기 때문에 ‘Allow the connection’을 선택합니다.

기본값(모두 선택)을 선택합니다.

룰의 이름 및 상세정보를 입력합니다. 원하는 이름으로 적어주시면 됩니다.

룰이 추가가 된 것을 확인할 수 있습니다. 왼쪽의 녹색원이 켜져 있어야 룰이 적용(Enable)이 된 것입니다.

자.. 다시 한번 체크를 해보겠습니다.
하지만 윈도우 방화벽을 열었음에도 아직 통신이 되지 않습니다 ㅠ. 다른 문제가 있는것 같습니다.

  1. azure trouble shoot 활용하기
    아마 Azure 네트워크상의 막히는 구간이 존재하는것 같은 느낌이 듭니다. nsg룰이 1,2개면 어느 룰에 의해서 막혔는지 간단하게 찾을 수 있지만
    실제 운영을 하다보면 룰이 100개 이상이 되는 경우도 존재합니다. 이럴때에는 Azure Portal에서 Trouble Shoot이라는 기능을 활용하면 좋습니다.

Azure Portal에서 VM선택 > Connection troubleshoot > Use Network Watcher for detailed connectio tracing

Specify manually를 선택후에 Target IP(10.0.0.5)를 입력합니다. 아래에 포트도 8080을 입력합니다. ‘Check’버튼 클릭.

결과가 나왔습니다. 상태가 Unreachable이네요.. 아래의 원인을 보면 10.0.0.4의 status에 문제가 있네요. 클릭을 해보았습니다.
8080-outbound-allow라는 룰에 의해 막혀있다고 나옵니다.

  1. azure nsg 확인하기
    위에서 막혔다고 표시된 룰을 확인하러 NSG에 들어옵니다.
    Allow를 해야 하는데 실수로 Deny를 한것 같습니다. ㅠ Allow로 수정해줍니다.

다시 상태를 확인하니 이번에는 연결에 성공하였습니다.!!!
