Azure VM Copy

2020-12-12
2020-12-12
Azure, Cloud, VM

이번 포스트는 Azure VM 복제입니다.
클라우드 환경에서는 자유롭게 vm을 생성할 수 있습니다.
그렇다면 기존에 셋팅한 vm을 복제하는 것은 어떻게 할 수 있을까요?
여러가지 방법이 있겠지만 이번 포스트에서는 Azure Snapshot을 사용하는 방법을 알려드리겠습니다.

이 방법은 제가 주로 사용하는 방법입니다.
VM의 osDisk와 DataDisk를 snapshot으로 만든후에 그걸 다시 disk로 만든다. osdisk를 기반으로 vm을 create 하면 됩니다.

  1. 복제하려는 VM의 OS Disk를 들어갑니다. 아래의 화면을 보시면 위쪽에 ‘+Create Snapshot’이 보이실 겁니다. 이 버튼을 클릭합니다.
  1. Snapshot을 만드는 화면으로 이동합니다. 이름과 Storage Type을 선택하고 나머지는 기본값으로 설정하겠습니다. Storage type은 HDD를 선택해도 나중에 Disk 생성후에 Disk 자체에서도 변경할 수 있습니다.
  1. 아래와 같이 snapshot이 만들어졌습니다. snapshot은 마찬가지로 Disk로도 바로 만들 수 있습니다.
    상단의 ‘+Create Disk’ 버튼을 클릭합니다.
  1. VM을 만드는 화면으로 이동합니다. 아마 이 화면은 친숙하실 겁니다.
    이름과 사이즈를 선택합니다. 사이즈 선택시에 기본값이 Premium SSD로 선택되어 있는데 저는 osDisk는 Standard SSD로 선택해서 사용합니다. Disk 비용을 절감할 수 있습니다.
  1. Disk가 완성이 되었습니다. 포털에서 osDisk를 바로 VM으로 만들 수 있도록 지원해주고 있습니다.
    ‘+Create VM’ 버튼을 클릭합니다. vm만드는 화면은 따로 설명은 하지 않겠습니다.
  1. VM이 완성이 되었습니다. public IP를 기억합니다.
  1. Rdp로 해당 vm에 접속하려고 하면 이미 기존의 호스트이름이 박혀 있는것을 확인할 수 있습니다. 이를 통해 해당 vm이 복제가 잘 되었다고 판단할 수 있을 것 같습니다.
